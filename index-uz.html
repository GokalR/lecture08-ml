Xossalar

Muammo: nazoratsiz o'qish

Pythonning oddiy ochiq atributi qiymatni "qanday bo'lsa shundayligicha" qaytaradi. O'qishda xotiradan qiymat oddiy qaytariladi. Tekshiruv, jurnalga yozish yoki qayta hisoblashni joylashtirib bo'lmaydi.

class CreditCard: def __init__(self, pan: str): self.pan = pan # обычный атрибут card = CreditCard("8600112233445566") print(card.pan) # весь номер виден целиком
8600112233445566

card.pan ni o'qiganda raqamning faqat bir qismini ko'rsatish istardik. Lekin oddiy atribut buni bajara olmaydi: u kod ishga tushirmaydi.

Xossalar

Yechim: @property dekoratori

@property metodni "virtual maydon"ga aylantiradi. Tashqaridan obyektni qavslarsiz, atribut sifatida o'qiydilar. Lekin Python aslida metodni chaqiradi va metod ichida istalgan kodni bajarish mumkin.

class CreditCard: def __init__(self, pan: str): self._pan = pan # _pan - внутреннее хранилище @property def pan(self) -> str: # это метод... return self._pan card = CreditCard("8600112233445566") print(card.pan) # ...но читаем без скобок, как поле

Analogiya: ombor kalitining o'rniga (maydonga to'g'ridan-to'g'ri kirish) biz berish oynasini (metod) o'rnatamiz. Tashqaridan bir xil qulay, ichkarida hammasi nazorat ostida.

Xossalar

Nima uchun atributni xossaga aylantirish kerak

Faqat qiymatni qaytaradigan getter o'z-o'zicha foydasiz. Xossa maydonga murojaat qilganda biror ish bajarilishi kerak bo'lganda zarur bo'ladi. Uchta tipik sabab:

1

O'qishda mantiq

Maydonga har murojaatda kod bajariladi: maskalash, jurnalga yozish, tekshirish.

2

Hisoblanadigan qiymatlar

Qiymat saqlanmaydi, balki obyektning dolzarb ma'lumotlari bo'yicha tezda hisoblanadi.

3

Interfeys moslashuvchanligi

Ochiq atribut tashqi kodga o'zgarishsiz xossaga aylanadi.

Xossalar · 1-sabab

O'qishda mantiq

Xossaga kod qo'shamiz. Har bir o'qishda u karta raqamini maskalaydi va murojaatni xavfsizlik jurnaliga yozadi.

class CreditCard: def __init__(self, client_name: str, raw_pan: str): self.client_name = client_name self._pan = raw_pan @property def masked_pan(self) -> str: print("[Аудит] Чтение номера карты") return f"{self._pan[:4]} **** **** {self._pan[-4:]}" card = CreditCard("Arman", "8600112233445566") print(f"Номер карты: {card.masked_pan}")
[Аудит] Чтение номера карты Номер карты: 8600 **** **** 5566

Xossalar · 2-sabab

Hisoblanadigan xossalar

Xossa qiymatni saqlamaydi, har murojaatda obyektning dolzarb ma'lumotlari bo'yicha qayta hisoblaydi. Ma'lumotlar o'zgarsa, natija ham o'zgaradi.

class LoanPortfolio: def __init__(self, principal: float, interest_rate: float): self.principal = principal self.interest_rate = interest_rate @property def total_debt(self) -> float: """Пересчитывается при каждом обращении""" return self.principal * (1 + self.interest_rate) portfolio = LoanPortfolio(10_000_000, 0.20) print(f"Итого к возврату: {portfolio.total_debt} сум") portfolio.principal -= 2_000_000 print(f"После погашения части: {portfolio.total_debt} сум")
Итого к возврату: 12000000.0 сум После погашения части: 9600000.0 сум

Pythonda qattiq maxfiylik yo'q: _pan maydoniga to'g'ridan-to'g'ri yetib borish va hatto uni o'zgartirish ham mumkin. _ ostidagi chiziq texnik jihatdan hech narsani taqiqlamaydi.

Ostidagi chiziq _ qulf emas, kelishuv sifatida ishlaydi. U: "maydon ichki, tashqaridan tegmang" deydi. Lekin jismonan kirish ochiq.

Xossalar · Savol tahlili

Getter ichki tuzilmani yashiradi

Tasavvur qiling: yarim yildan keyin qoidalar o'zgardi. Karta raqamini endi ochiq saqlash mumkin emas, u shifrlanadi. Sinfning ichki tuzilmasini o'zgartiramiz:

# Было: номер хранится как есть self._pan = raw_pan # Стало: номер хранится зашифрованным self._encrypted_pan = encrypt(raw_pan) # Геттер меняем в одном месте, имя свойства прежнее: @property def masked_pan(self) -> str: real = decrypt(self._encrypted_pan) return f"{real[:4]} **** **** {real[-4:]}"

card.masked_pan ni o'qiyotgan kod hech qanday tuzatishsiz ishlashda davom etadi. card._pan ni to'g'ridan-to'g'ri olgan kod hamma joyda buziladi: maydon endi yo'q.

Xossalar

Yozishni nazorat qilish: setter

Hozircha @property faqat o'qishni boshqaradi. Yozishni, ya'ni = orqali qiymat berishni ushlab olish uchun xossaga setter qo'shiladi: @nom.setter dekoratorli metod.

class Account: def __init__(self, balance: float): self._balance = balance @property def balance(self): # чтение: acc.balance return self._balance @balance.setter def balance(self, value): # запись: acc.balance = ... self._balance = value

Getter va setter bir xil balance nomini olib yuradi. Endi maydonni ham o'qish, ham unga qiymat berish mumkin, lekin yozish metod orqali o'tadi.

Xossalar

Ma'lumotlarni tekshiruvchi setter

Setterning asosiy foydasi: qiymatni saqlashdan oldin tekshirish. Agar u nomaqbul bo'lsa, setter xato ko'taradi va maydon avvalgicha qoladi.

class CreditLimitManager: def __init__(self, current_limit: float): self._limit = current_limit @property def limit(self) -> float: return self._limit @limit.setter def limit(self, new_value: float): if new_value < 0: raise ValueError("Лимит не может быть отрицательным") self._limit = new_value manager = CreditLimitManager(50_000_000) try: manager.limit = -100 except ValueError as error: print(f"Запись отклонена: {error}")
Запись отклонена: Лимит не может быть отрицательным

Ichki tuzilma

Getter va setter: bitta xossa

Getter va setter birgalikda bitta xossani tashkil qiladi. Python o'zi kerakli metodni tanlaydi: maydonni o'qiyotganimizmi yoki unga qiymat berayotganimizmi.

@property создаёт свойство 'limit' с логикой чтения @limit.setter к тому же свойству добавляет логику записи manager.limit Python вызывает геттер manager.limit = 700000 Python вызывает сеттер со значением 700000

Getter va setter nomlari mos kelishi shart. Shuning uchun dekorator @limit.setter shaklida yoziladi: u allaqachon yaratilgan limit xossasiga bog'lanadi.

class Contract: def __init__(self, contract_id: str, client: str): self._contract_id = contract_id self.client = client @property def contract_id(self) -> str: return self._contract_id # сеттера нет contract = Contract("UZ-2024-001", "Тимур") print(contract.contract_id) try: contract.contract_id = "UZ-2024-999" except AttributeError: print("Защита сработала: номер контракта менять нельзя")

Sinf darajasi

Nima uchun @classmethod kerak

Sinfda bitta konstruktor bor - __init__ metodi. U joylariga taqsimlangan tayyor maydonlarni kutadi. Lekin ma'lumotlar ko'pincha boshqa shaklda keladi: APIdan JSON, CSV-fayldan satr, oddiy hozirgi vaqt.

import json # Ответ API пришёл одной JSON-строкой: raw = '{"amount": 150000, "currency": "UZS", "ts": "2024-01-15"}' # Перед созданием объекта строку приходится разбирать вручную: data = json.loads(raw) t = Transaction(data["amount"], data["currency"], data["ts"])

Agar obyekt loyihaning o'n joyida shu tariqa yaratilsa, JSONni tahlil qilish o'n marta takrorlanadi. Yaratish mantig'i kod bo'ylab "yoyilib ketadi".

Sinf darajasi

Yechim: @classmethod

@classmethod dekoratorli metod ikkinchi, nomlangan konstruktor sifatida ishlaydi. U birinchi argumentda self obyektini emas, balki cls sinfining o'zini oladi.

Oddiy metod

self ni oladi

Metodga aniq obyekt keladi. Metod uning ma'lumotlari bilan ishlaydi, masalan self.amount.

Sinf metodi

cls ni oladi

Metodga sinfning o'zi keladi. cls(...) chaqiruvi Transaction(...) kabi yangi obyekt yaratadi.

Ma'lumotlarni tahlil qilish mantig'i sinf metodi ichiga yashiriladi. Tashqarida qisqa chaqiruv qoladi: Transaction.from_json(raw).

Sinf darajasi

Fabrika metodi: JSONdan

Yechimni kodda yig'amiz. from_json metodi satrni o'z ichida tahlil qiladi va cls(...) orqali tayyor obyektni qaytaradi.

import json class Transaction: def __init__(self, amount: float, currency: str, timestamp: str): self.amount = amount self.currency = currency self.timestamp = timestamp @classmethod def from_json(cls, json_str: str) -> 'Transaction': data = json.loads(json_str) # разбираем ответ API return cls(data["amount"], data["currency"], data["ts"]) def __str__(self): return f"{self.amount} {self.currency} @ {self.timestamp}" t1 = Transaction.from_json('{"amount": 150000, "currency": "UZS", "ts": "2024-01-15"}') print(t1)
150000 UZS @ 2024-01-15

@classmethod cls ni oladi va cls(...) orqali obyekt yaratadi - bu Transactionni yig'ishning ikkinchi usuli.

Sinf darajasi

CSV va vaqtdan konstruktorlar

Shu Transaction sinfining yana ikkita fabrika metodi:

@classmethod def from_csv_row(cls, csv_line: str) -> 'Transaction': parts = csv_line.strip().split(",") # из строки CSV return cls(float(parts[0]), parts[1], parts[2]) @classmethod def create_now(cls, amount: float) -> 'Transaction': now = datetime.now().isoformat() # текущее время return cls(amount, "UZS", now) t2 = Transaction.from_csv_row("320000.0,UZS,2024-01-15T11:00:00") t3 = Transaction.create_now(75000.0) print(t2) print(t3)

Bitta sinf - bir nechta kirish nuqtasi: JSONdan, CSVdan, hozirgi vaqtdan.

Sinf darajasi

Merosda cls va qattiq sinf nomi

Fabrika metodi ichida obyekt sinf nomi orqali emas, balki cls(...) orqali yaratiladi. Shunda metod voris sinflarda ham to'g'ri ishlaydi.

class BaseCard: def __init__(self, number: str): self.number = number @classmethod def from_masked(cls, masked: str) -> 'BaseCard': fake = masked.replace("****", "0000") return cls(fake) # cls, а не BaseCard - иначе потомки сломаются def __str__(self): return f"{self.__class__.__name__}: {self.number}" class GoldCard(BaseCard): pass print(BaseCard.from_masked("8600 **** **** 5566")) print(GoldCard.from_masked("8600 **** **** 5566"))
BaseCard: 8600 0000 0000 5566 GoldCard: 8600 0000 0000 5566

cls(...) o'rniga BaseCard(...) bo'lganida gold obyekti GoldCard emas, BaseCard nusxasi bo'lib qolardi.

Sinf darajasi

Nima uchun @staticmethod kerak

Shunday funksiya bo'ladiki, u mantiqan sinfga tegishli, lekin na aniq obyekt bilan, na sinfning o'zi bilan ishlamaydi. Unga na self, na cls kerak.

Misol: PhoneValidator sinfi yonida telefon raqami formatini tekshirish. Bu sof yordamchi vosita: satr oladi, satr qaytaradi. Uni oddiy funksiyaga chiqarish mumkin, lekin shunda u tegishli bo'lgan sinf bilan aloqasi yo'qoladi.

@staticmethod dekoratori shunday yordamchini sinf ichiga joylashtirishga imkon beradi. U yaqin kod yonida guruhlanadi, lekin na obyekt, na sinf oladi.

Sinf darajasi

Statik metodlar: @staticmethod

@staticmethod ga ega metod self va cls siz e'lon qilinadi. U sinf orqali chaqiriladi. Bu yerda u telefon raqamini yagona formatga keltiruvchi yordamchi vosita.

class PhoneValidator: @staticmethod def clean_phone(phone_str: str) -> str: """Приводит номер к стандартному формату""" cleaned = "".join(c for c in phone_str if c.isdigit()) if cleaned.startswith("8"): cleaned = "7" + cleaned[1:] return f"+{cleaned}" formatted = PhoneValidator.clean_phone("8 (90) 123-45-67") print(f"Результат: {formatted}")
Результат: +79012345667

Nima uchun shunchaki global funksiya emas?

Вопрос 1: Нужен доступ к данным конкретного объекта? (self.balance, self.owner, self._pin) ДА → обычный метод: def method(self, ...) Protokollar

Protokollar va sehrli metodlar

Dunder metodlar (Double Underscoredan) foydalanuvchi obyektlarini Pythonning standart operatorlari va funksiyalari bilan integratsiya qilishga imkon beradi.

__add__, __eq__, __len__ va boshqalarni qayta aniqlab, siz obyektlar qanday qo'shilishini, taqqoslanishini va tilning standart konstruksiyalarida qanday ishlashini belgilaysiz.

Yakunlar

7-leksiya yakunlari

  • Polimorfizm - turli amalga oshirishlar uchun yagona interfeys; bog'liq sinflar uchun qayta aniqlash, mustaqillar uchun Duck Typing
  • MRO - C3-linearizatsiya algoritmi ko'p merosda noaniqlikni bartaraf etadi
  • Inkapsulyatsiya - _, __, @property va setterlar ma'lumotlarni himoya qiladi va kirishni nazorat qiladi
  • @classmethod - sinfga bir nechta kirish nuqtasi, cls orqali merosda to'g'ri ishlaydi
  • @staticmethod - sinf ichida guruhlangan izolyatsiyalangan yordamchi vositalar
  • Dunder metodlar - obyektlarni Pythonning standart operatorlari bilan integratsiya qilish
